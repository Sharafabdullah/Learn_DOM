DOM NodeList

A NodeList object is a list (collection) of nodes extracted from a document. Understanding the differences between NodeList and HTMLCollection is crucial for effective DOM manipulation.

NodeList Characteristics

NodeList objects have several important features:

  • Array-like access with index numbers
  • length property for counting nodes
  • Access nodes by index: nodeList[0]
  • Can be iterated with for loops
  • Contains all types of nodes (element, text, comment)

Static vs Live NodeLists

NodeLists can be either static or live:

  • querySelectorAll() - Static NodeList
  • childNodes - Live NodeList
  • Static = snapshot at creation time
  • Live = updates with DOM changes

NodeList vs HTMLCollection

Key differences between NodeList and HTMLCollection:

Feature NodeList HTMLCollection
Content All node types Element nodes only
Access Methods Index only Index, name, id
querySelectorAll() Static NodeList N/A
getElementsByTagName() N/A Live HTMLCollection
childNodes Live NodeList N/A
children N/A Live HTMLCollection

Best Practices

  • Use querySelectorAll() for static selections
  • Be aware of live vs static behavior
  • Consider children instead of childNodes for elements only
  • Convert to array with Array.from() for array methods
  • Use forEach() method available on NodeList (modern browsers)
  • Cache length in loops for better performance
